<task type="auto">
  <name>Task 1: Classify all gaps and propose fixes with effort estimates</name>
  <files>audit/reports/12-V1-CERTIFICATION.md</files>
  <action>
Read the gap analysis report (12-GAP-ANALYSIS.md) and classify every gap using this severity framework:

**Critical** — Blocks V1 certification. Core assumption violated, claim fundamentally wrong, or functionality broken.
Criteria: Would a user relying on the documented behavior experience failure?

**Important** — Should fix before V1.1. Significant gap but workarounds exist or impact is limited.
Criteria: User can work around it but experience is degraded or claim needs revision.

**Enhancement** — Nice-to-have for V2+. Minor polish or optimization.
Criteria: User unlikely to notice, or improvement is incremental.

For each gap, document:
1. **GAP-ID** (from 12-GAP-ANALYSIS.md)
2. **Severity**: Critical / Important / Enhancement
3. **Classification Rationale**: Why this severity level (1-2 sentences)
4. **Proposed Fix**: Specific implementation change (file, function, what to change)
5. **Fix Location**: Exact file path and approximate line range
6. **Effort Estimate**: Lines of code + time (e.g., "~15 lines, 10 min")
7. **Target Release**: V1 (Phase 13) / V1.1 / V2
8. **Dependencies**: Other gaps that must be fixed first (if any)

Create a summary table at the top:

| Severity | Count | Phase 13 Action |
|----------|-------|-----------------|
| Critical | N | Must fix before V1 ship |
| Important | N | Fix in V1.1 (document in release notes) |
| Enhancement | N | Track for V2 |

Group gap classifications by target release:
- **Phase 13 Scope** (Critical fixes): List exact gaps, ordered by dependency
- **V1.1 Scope** (Important): List with effort estimates, priority order
- **V2+ Scope** (Enhancement): List with brief rationale for deferral

Cross-reference with Phase 10 deferred categorization (10-03) to ensure consistency — V1.1-Important items from Phase 10 should align with Important gaps here.

Cross-reference with Phase 11 protocol gap recommendations to ensure those 3 fixes are included.
  </action>
  <verify>Count gaps by severity. Verify every gap from 12-GAP-ANALYSIS.md has a classification. Verify Phase 13 scope is actionable (specific files, specific changes).</verify>
  <done>All gaps classified with severity, fix proposals, effort estimates. Phase 13 scope clearly defined. V1.1 and V2 scopes documented.</done>
</task>
